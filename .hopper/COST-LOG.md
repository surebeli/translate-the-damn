## Issues hit

- 2026-06-18: `hopper-dispatch` on PATH was a stale v0.6.1 shim (no mimo). Filed `.hopper/ISSUE-stale-dispatch-binary.md`; workaround = absolute-path 0.12.0 binary; fix applied = repoint shim → 0.12.0 plugin binary (backup at `~/.local/bin/hopper-dispatch.bak.v0.6.1-2026-06-18`).
- mimo known 180s `code-impl` timeout (see plugin `ISSUE-mimo-codeimpl-timeout.md`) → route bulk code-impl away from mimo; mimo reserved for review (extended floor) + small/reasoning edits.
- **Sandbox auto-detection gotcha**: dispatch sets `--sandbox read-only` if the spec text contains "read-only" anywhere (even describing a reference file). kimi ignores the flag (edits land); opencode/codex HONOR it (would block edits) ⇒ for code-impl to opencode, pass `--sandbox danger-full-access` explicitly. (For genuine review tasks, read-only is correct.)
- **Background Bash cwd**: background Bash processes don't inherit the session cwd ⇒ relative `.hopper/` paths fail. ⇒ always `export HOPPER_DIR=<repo>/.hopper` + use absolute paths in background dispatch Bash.
- **opencode model-cache warning**: `hopper-dispatch --probe opencode` populates the model cache (silences the "no probed model cache" warning + validates `--model`). Optional; models work without it.
- **Vendor auto-commit inconsistency**: kimi committed for T-MAC-11 (`11e1b62`) but NOT for T-MAC-10/12 (cited AGENTS.md); opencode didn't commit. ⇒ manage commits at milestones, don't rely on vendors.
- **queue.md Depends range notation invalid**: `T-MAC-10..16` is parsed as ONE literal dependency ID ⇒ `Task not eligible: dependency T-MAC-10..16 not found`. Use comma-separated IDs (or empty). Fixed T-MAC-20/36/40 (2026-06-18).
- **Background Bash must detect dispatch failure**: T-MAC-20's first dispatch errored at start ("Task not eligible") but the poll loop still ran 120×10s = 20min of wasted polling (no output.md ever appeared). ⇒ capture dispatch output, check for `BACKGROUND DISPATCH STARTED` before polling; abort on failure. Fixed in the re-dispatch pattern.
- **queue.md Brief cell `|` breaks the v2 table parser**: T-MAC-36's Brief had `(Clipboard|Hotkey)` — the literal `|` split the markdown row, so the Vendor column was mis-parsed as a Brief fragment ⇒ `No vendor adapter registered for 'Hotkey)→Popup…'`. ⇒ never put a raw `|` in a Brief cell (use `/` or `,`). Fixed T-MAC-36 (`(Clipboard/Hotkey)`) 2026-06-18.
